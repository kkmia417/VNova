# Architecture Document Index

Status: Review map; protected human decisions remain binding

This index prevents planning and review material from being mistaken for accepted architecture.
`AGENTS.md` and accepted ADRs remain authoritative. `Draft`, `Proposed`, `OPEN`, `Active planning
sequence`, and `Normative target` are document classifications, not acceptance states. Unless an
accepted ADR or recorded protected human decision explicitly adopts a document, it is review
evidence only and cannot authorize runtime implementation.

## Boundary Terminology

The historical handoff refers to five conceptual planes but does not provide their names,
responsibilities, or dependency directions. OD-018 requires a human to recover that taxonomy or
retire the phrase. Until then, reviewers and tooling must use the explicit deployed surfaces,
protected packages, responsibility boundaries, and dependency rules in the system overview and
ADR-001. No unnamed plane is an implementation boundary.

## Review Rule

For the governance bootstrap, start with the exact
[Stage A handoff](../governance/foundation-stage-a-review-handoff.md). For the separate executable
foundation candidate and any broader protected architecture surface, read the
[Stage B architecture foundation review packet](../governance/architecture-foundation-review.md),
then use the [feature architecture review packet](../governance/feature-architecture-review.md)
for the named capability. Closing the foundation gate does not close a feature-specific ADR gate,
and local verification does not substitute for protected human review, remote CI, or repository
Ruleset enforcement. A feature that passes architecture review still requires the
[operational readiness review](../governance/operational-readiness-review.md) before live use.
